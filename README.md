# Garden Shop Project

Это веб-приложение для магазина товаров для сада, разработанное с использованием современного стека технологий.

## О проекте

Приложение представляет собой интернет-магазин с возможностью просмотра товаров, фильтрации по категориям, добавления товаров в корзину и оформления заказа.

### Основные функции:
- **Главная страница**: Промо-блок, категории, новинки.
- **Каталог**: Просмотр всех товаров и товаров по категориям.
- **Корзина**: Добавление/удаление товаров, изменение количества, подсчет общей суммы.
- **Оформление заказа**: Валидация данных (имя, телефон, email) и отправка заказа.
- **Адаптивность**: Приложение корректно отображается на различных устройствах.

## Техническое оснащение и Архитектура

### Стек технологий:
- **Frontend**:
  - [React](https://reactjs.org/) (v18) - библиотека для построения пользовательских интерфейсов.
  - [TypeScript](https://www.typescriptlang.org/) - строгая типизация для повышения надежности кода.
  - [Vite](https://vitejs.dev/) - быстрый сборщик и dev-server.
  - [Tailwind CSS](https://tailwindcss.com/) - utility-first CSS фреймворк для стилизации.
  - [React Router](https://reactrouter.com/) - маршрутизация в SPA.
  - [Axios](https://axios-http.com/) - HTTP-клиент для запросов к API.

- **Backend**:
  - [Node.js](https://nodejs.org/) & [Express](https://expressjs.com/) - серверная часть (находится в папке `telran-backend`).
  - [SQLite](https://www.sqlite.org/) + [Sequelize](https://sequelize.org/) - база данных и ORM.

### Архитектурные решения:
1.  **Unified Development**: Проект настроен на запуск одной командой (`npm run dev`), которая параллельно поднимает Backend (порт 3334) и Frontend (порт 8080).
2.  **Proxy Configuration**: В `vite.config.ts` настроен прокси. Все запросы на `/categories`, `/products`, `/sale`, `/order` и статику (`/category_img`, `/product_img`) автоматически перенаправляются на бэкенд. Это решает проблему CORS и упрощает конфигурацию API на клиенте (используются относительные пути).
3.  **Feature-Based Structure**:
    - `client/pages`: Компоненты страниц (маршрутов).
    - `client/components`: Переиспользуемые UI компоненты.
    - `client/context`: Глобальное состояние (CartContext) для управления корзиной.
    - `client/api.ts`: Слой абстракции для взаимодействия с API.
4.  **Стилизация**: Использована дизайн-система на базе Tailwind CSS. Кастомные цвета и шрифты настроены в `tailwind.config.ts` согласно макету.
5.  **State Management**: Для корзины используется React Context API + LocalStorage для сохранения данных между сессиями.

## Как запустить

Для запуска приложения вам потребуется [Node.js](https://nodejs.org/).

1.  **Установка зависимостей**:
    Одной командой устанавливаются зависимости для всего проекта (клиент и бэкенд уже преднастроены).
    ```bash
    npm install
    # Также установите зависимости для бэкенда, если это первый запуск:
    cd telran-backend && npm install && cd ..
    ```

2.  **Запуск в режиме разработки**:
    ```bash
    npm run dev
    ```
    Эта команда запустит и сервер, и клиент.
    - Приложение будет доступно по адресу: [http://localhost:8080](http://localhost:8080)

3.  **Сборка для продакшена**:
    ```bash
    npm run build
    ```

## Критерии выполнения

- **Паттерн архитектуры**: Разделение на клиент и сервер, компонентный подход React, использование хуков и контекста.
- **Стилизация**: Полное соответствие макету (шрифты, цвета, отступы, сетка товаров), адаптивная верстка.
- **Техническое оснащение**:
    - Валидация форм.
    - Динамический бейдж корзины.
    - Оптимизированная загрузка изображений.
    - SPA навигация без перезагрузки страницы.
